Unsupervised Machine Learning
--- Page 6 ---
Unsupervised Learning A machine learning technique that works with unlabeled data, identifying patterns and structures within it and lacks a specific output or prediction goal based on labeled training data Goals: • Explore the types or heterogeneity within the data • Find natural groupings, or clusters, in the data that reveal underlying structures
--- Page 7 ---
Key Algorithms: K - means Clustering Overview • Bins data into N groups, buckets or clusters based on the similarity of characteristics • Partitions the data into clusters • Assigns each data point to the cluster with the nearest mean or average Process • Starts by selecting K initial cluster centers • Assigns iteratively each data point to the nearest cluster centre • Recalculates cluster centres based on assigned data points • Repeats the process until the cluster centres stabilises Application K - means is a widely used tool for customer segmentation, enabling businesses to group customers based on their purchasing behaviour , allowing for targeted marketing strategies
--- Page 8 ---
Key Algorithms: Gaussian Mixture Models (GMM) Overview • Uses a probabilistic approach • Assumes data is generated from a mixture of several Gaussian distributions • Assigns probabilities to each data point for belonging to each cluster, allowing for soft clustering Application • Used in scenarios with overlapping clusters to capture uncertainty in clustered assignments • Used in tasks like image segmentation and anomaly detection

Supervised Learning Methods
--- Page 12 ---
Supervised Learning • Uses input data along with an explicit output variable, referred to as labelling • Involves training the algorithm on a dataset that includes both the input features and corresponding correct outputs Key Aspects Process: Algorithm • Receives label data in which observations are tagged with one or more labels • Learns the mapping between inputs and outputs with the help of these labels • The goal of supervised learning is to use training data to learn the relationship between input and output, enabling accurate predictions when presented with new, unseen data.
--- Page 13 ---
Supervised Learning • Supervised learning aids in prediction, helping managers understand key factors and their importance, guiding decision - making and strategy formulation • Essential for tasks that involve prediction or predicting outcomes based on historical data • Facilitates understanding relationships between variables and making informed predictions, crucial for strategic business decisions Importance Applications • Predicting future outcome: Using past data to make predictions about the future or unknown data • Making informed decisions: Helping managers make informed decisions about resource allocations in advertising and making strategies around advertising

Supervised Learning Methods Part II
--- Page 15 ---
Supervised Learning: Regressions Linear Regression • Used when the target variable ‘y’, takes a range of continuous values • Aimed at modeling the relationship between the input variables and the continuous output variable Logistic Regression • Works best when the target variable ‘y’ takes only two possible values • Includes categories such as yes, no, true, false, or gender • Predicts the possibility of the target variable falling into one of the categories
--- Page 16 ---
Tree - Based Algorithms • Splits the data into subsets based on values of the input features • Each internal node represents a decision based on a particular feature or variable • Each branch represents the outcome of the decision • Each leaf node represents a class label or continuous value
--- Page 17 ---
Tree - Based Algorithms: Advantages Easy to understand and interpret Capable of capturing non - linear relationships between features and the target variable Capable of being used for classification, where 'y' ranges between zero and one
--- Page 18 ---
Neural Networks Neural Network Basics: • Model data by mimicking the human brain • Consist of layers of interconnected nodes that process input data • Learn to recognise patterns through training • Neural Networks: Feed Forward Data flows unidirectionally from the input layer, through hidden layers, to the output layer, ensuring sequential processing by each layer for the final output Neural Networks: Importance of Weights • A node assigns a weight that determines the relative importance of input signals • A node then multiples each input by its corresponding weights and adds the resulting products together, yielding a single number
--- Page 19 ---
Training in Neural Networks • Training data is fed into the input layer, which then passes through the succeeding layers • As data moves through layers, it gets multiplied and added depending on the number of hidden layers. This continues until it reaches the output layer • During training phase, the neural network continuously adjusts weights and biases until it consistently produces outputs for the training data with the same kind of labels • The goal is to minimise the error in prediction
