## 11. AI Strategy Roadmap Details

### 11.1 AI Use-Case Prioritization Model

Use this framework in interviews:

| Dimension | What to Assess |
|---|---|
| Business value | Time saved, cost reduced, risk reduced, quality improved |
| Workflow frequency | Daily/weekly workflows before rare workflows |
| Data readiness | Structured data, accessible systems, quality of knowledge sources |
| Risk level | Privacy, compliance, financial impact, employee impact |
| Automation suitability | Human assist vs human approval vs full automation |
| Adoption feasibility | Existing workflow fit, training need, resistance level |
| Measurement clarity | Can outcomes be measured objectively? |

### 11.2 AI Governance Model

Use this language:

> For enterprise AI, I think in terms of boundaries: who can invoke the agent, what systems the agent can access, what actions it can execute, what data it can return, what needs human approval, and what evidence is captured for audit and learning.

Governance controls to mention:

- Role-based access control.
- Tool-level authorization.
- Data classification.
- Prompt and response logging.
- Audit trails.
- Evaluation datasets.
- Hallucination checks.
- Human approval for high-risk actions.
- Observability of agent decisions.
- Drift and quality monitoring.
- Incident response for AI failures.

### 11.3 AI ROI Metrics

Use these metrics:

| Outcome Area | Metrics |
|---|---|
| Engineering productivity | Lead time, PR cycle time, test generation effort, code review turnaround |
| Quality | Defect leakage, escaped defects, test coverage, rework reduction |
| Operations | MTTR, incident triage time, alert noise reduction |
| Cost | Cloud waste reduction, manual effort reduction, support cost reduction |
| Delivery | Sprint predictability, dependency closure, release frequency |
| Adoption | Active users, repeat usage, workflow coverage, human override rate |
| Safety | Policy violations, risky tool calls blocked, audit completeness |
